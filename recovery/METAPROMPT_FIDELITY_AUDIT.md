# Metaprompt Fidelity Audit

## Ruling

The concern is correct. The v0.1 repository often preserves the **outline** of a strong prompt while dropping the procedural depth that made the original effective: full source recovery, context composition, authority resolution, typed inputs and outputs, phased execution, validation, rollback, failure conditions and exact resumption.

Several public files are only about 1.1–2.4 KB, while recovered originals are commonly 16–36 KB. This is not harmless compression. It changes likely task performance and can turn a governed programme into a generic checklist.

## Required repository architecture

```text
prompts/
  compact/         thin launchers
  full/            canonical operational prompts
  source-originals/ preserved historical source editions
  fixtures/        evaluation cases and negative controls
  degradation/     known loss risks and compact-vs-full comparisons
```

Every prompt record should identify source file, version, line or artifact range, checksum, transformation method, reviewer, evaluation fixtures, known omissions and whether the compact form is safe for standalone execution.

## Publication rule

A compact prompt may point to and activate a full edition. It may not silently replace it. Where the exact original survives, preserve it verbatim and publish a separately reviewed, context-agnostic edition rather than bleaching away operative constraints.

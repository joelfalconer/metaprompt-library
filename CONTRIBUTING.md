# Contributing

A strong addition must solve a recurring class of work, not only one historical task.

## Acceptance questions

- Is the job likely to recur across domains or projects?
- Does the prompt define inputs, outputs, evidence rules, quality gates, and stop conditions?
- Can private or domain-specific nouns be replaced without weakening the method?
- Is there an existing prompt to revise or extend?
- Does it distinguish proposals from accepted state?
- Does it create a durable output rather than only conversational prose?
- Can success and failure be evaluated?

Use `templates/PROMPT_TEMPLATE.md` and update `registry/prompts.yaml`.

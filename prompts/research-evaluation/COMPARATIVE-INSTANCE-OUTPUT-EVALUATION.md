# Comparative Instance Output Evaluation

## Mission

Evaluate two or more outputs produced from similar but not necessarily identical prompts.

## Procedure

1. Establish prompt asymmetry, source access, model/tool differences and stopping conditions.
2. Reconstruct what each instance actually produced, not only its final prose.
3. Score prompt fidelity, coverage, ontology, ownership, safety, operationalization, human fit, interoperability, consistency and economy.
4. Identify best decisions, worst defects, missed requirements, unsupported claims, hidden overreach and valuable unique contributions.
5. Distinguish a weak result from a result solving a different layer of the problem.
6. Create a synthesis showing what each should borrow from the other.
7. Derive an improved metaprompt, system changes, evaluation fixtures and next vertical slice.

Every criticism must point to evidence. Avoid pseudo-precision: explain score meaning and confidence.

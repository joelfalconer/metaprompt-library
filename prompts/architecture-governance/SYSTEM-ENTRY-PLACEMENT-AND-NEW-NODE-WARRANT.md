# System Entry, Placement, and New-Node Warrant

## Mission

Before creating a repository, project, agent, database or documentation hub, map incoming work against the existing ecosystem.

## Procedure

1. Read the ecosystem entry point, current map, owner, repository, realm, capability, interface, lock and handoff registries.
2. Inventory the incoming sources and identify their teleology, objects, capabilities, authority needs, sensitivity and lifecycle.
3. Search for exact IDs, aliases, neighbouring scopes and similar projects.
4. Evaluate dispositions:
   - extend existing owner;
   - merge and migrate value;
   - donor/archive only;
   - local subproject or capability package;
   - quarantine pending evidence;
   - warrant a new node.
5. A new node requires a distinct durable owner, non-redundant teleology, lifecycle, interfaces, custody, operating burden and migration plan.
6. Produce placement docket, conflict audit, proposed paths, owner handoff, rollback and registry patch candidate.

Do not create infrastructure merely to give a chat somewhere to go.

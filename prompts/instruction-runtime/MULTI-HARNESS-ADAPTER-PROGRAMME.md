# Multi-Harness Adapter Programme

## Mission

Create a governed adapter programme that compiles shared instruction, capability and routing sources into platform-specific projections.

## Required outputs

- source and donor inventory;
- ownership ADR;
- canonical atom and capability registries;
- platform adapter manifests;
- collision and precedence tests;
- project starter profiles;
- degraded-mode rules;
- migration ledgers;
- validation fixtures;
- operator walkthrough;
- drift and rollback plan.

## Rules

- keep project-local rules sovereign;
- namespace discovered agents, skills, commands and plugins;
- do not share one late-loading config directory as a universal brain;
- separate portable intent from Windows, Linux, WSL, macOS and server bindings;
- keep secrets as references and runtime injection;
- distinguish documented, observed and inferred platform behavior;
- deploy only after effective-config verification.

# Global Agent Instructions Compiler Commission

## Mission

Design and package a global instruction file as a compiled projection from current canonical doctrine, not as a hand-maintained catalogue or incremental edit of stale rules.

## Required work

1. Read current constitutional, user, organization, repository and harness sources plus active locks.
2. Inventory donor global instructions and project-local examples.
3. Define instruction atoms with ID, scope, precedence, consequence, source revision, expiry and projection targets.
4. Decide which atoms belong globally, project-locally, in skills, in task prompts, or in deterministic enforcement.
5. Draft a concise global projection that teaches orientation, source-of-truth lookup, tool discipline, local-rule precedence, privacy, mutation gates, receipts and exact resumption.
6. Produce global-versus-project routing matrix, compiler architecture, validation fixtures, drift detection and rollback.
7. Do not mutate live configuration without a separate migration work order.

The global file should point to discoverable deeper context rather than importing the whole institution into every session.
